Escribe Código R Como un Profesional

Las buenas prácticas no solo hacen que tu código sea más legible y mantenible, sino que también te ayudan a evitar errores y trabajar más eficientemente. Aprende los estándares que usan los profesionales de R.

¿Por qué importan las buenas prácticas? Un código bien escrito es más fácil de entender, debuggear, compartir y mantener. Te ahorrará tiempo y dolores de cabeza.
Nombres de Variables Descriptivos

Los nombres de variables deben explicar claramente qué contienen. Tu futuro yo te lo agradecerá.

❌ Mal ejemplo
# Nombres confusos
x <- 25
y <- c(160, 170, 175, 168)
z <- mean(y)
a <- y> z
df <- data.frame(y, a)

Problema: Imposible entender qué hace el código sin leer todo.

✅ Buen ejemplo
# Nombres descriptivos
edad_estudiante <- 25
estaturas_cm <- c(160, 170, 175, 168)
estatura_promedio <- mean(estaturas_cm)
es_alto <- estaturas_cm> estatura_promedio
datos_estudiantes <- data.frame(estaturas_cm, es_alto)

Ventaja: El código se lee como una historia clara.

Convenciones de Nombres en R:
  • snake_case: mi_variable_larga (recomendado)
  • camelCase: miVariableLarga (alternativo)
  • Evitar: mi.variable (puede confundirse con métodos)
  • Constantes: PI_VALOR <- 3.14159 (mayúsculas)
Comentarios Inteligentes

Los comentarios deben explicar el "por qué", no el "qué". El código ya dice qué hace.

❌ Comentarios inútiles
# Crear variable edad
edad <- 25

# Sumar 1 a edad
edad <- edad + 1

# Imprimir edad
print(edad)

Problema: Los comentarios no añaden información útil.

✅ Comentarios útiles
# Datos del paciente al momento del estudio
edad_actual <- 25

# Ajuste por tiempo transcurrido desde el estudio
edad_ajustada <- edad_actual + 1

# Verificar que el cálculo es correcto
print(edad_ajustada)

Ventaja: Explica el contexto y las decisiones.

Tipos de Comentarios Útiles:
  • Contexto: ¿Por qué este análisis es importante?
  • Decisiones: ¿Por qué elegiste este método?
  • Warnings: ¿Qué tener cuidado al modificar?
  • TODOs: ¿Qué falta por implementar?
  • Fuentes: ¿De dónde vienen los datos o fórmulas?
Estructura y Organización

Un código bien organizado es más fácil de seguir y mantener. Sigue una estructura lógica.

✅ Estructura Recomendada
################################################################################
# Título: Análisis de Satisfacción del Cliente
# Autor: Tu Nombre
# Fecha: 2024-01-15
# Descripción: Análisis exploratorio de encuesta de satisfacción
################################################################################

# 1. CONFIGURACIÓN INICIAL ----
library(dplyr)
library(ggplot2)
library(readxl)

# Limpiar entorno
rm(list = ls())

# Configurar directorio
setwd("~/proyecto_satisfaccion")

# 2. CARGAR Y LIMPIAR DATOS ----
datos_raw <- read_excel("encuesta_satisfaccion.xlsx")

# Limpiar nombres de columnas
nombres_limpios <- c("id", "edad" , "satisfaccion" , "recomendaria" )
names(datos_raw) <- nombres_limpios

# 3. ANÁLISIS EXPLORATORIO ----
summary(datos_raw)

# 4. VISUALIZACIONES ----
grafico_satisfaccion <- ggplot(datos_raw, aes(x=satisfaccion)) +
geom_histogram(bins = 10, fill = "skyblue") +
labs(title = "Distribución de Satisfacción del Cliente")

# 5. RESULTADOS Y EXPORTAR ----
ggsave("satisfaccion_distribucion.png", grafico_satisfaccion)
Principios de Organización:
  • Secciones claras: Usa comentarios con ---- para dividir
  • Orden lógico: Setup → Datos → Análisis → Resultados
  • Espaciado: Líneas en blanco para separar bloques lógicos
  • Límite de línea: Máximo 80 caracteres por línea
Crear Funciones Reutilizables

Si repites código más de 2 veces, convértelo en una función. Principio DRY: Don't Repeat Yourself.

❌ Código repetitivo
# Calcular IMC para 3 personas (repetitivo)
peso1 <- 70; altura1 <- 1.75
imc1 <- peso1 / (altura1^2)
cat("Persona 1 - IMC:", round(imc1, 2))

peso2 <- 65; altura2 <- 1.68
imc2 <- peso2 / (altura2^2)
cat("Persona 2 - IMC:", round(imc2, 2))

peso3 <- 80; altura3 <- 1.82
imc3 <- peso3 / (altura3^2)
cat("Persona 3 - IMC:", round(imc3, 2))
✅ Función reutilizable
# Crear función para calcular IMC
calcular_imc <- function(peso, altura, nombre=NULL) {
# Validar inputs
if (peso <= 0 || altura <=0) {
stop("Peso y altura deben ser positivos")
}

# Calcular IMC
imc <- peso / (altura^2)

# Mostrar resultado
if (!is.null(nombre)) {
cat(nombre, "- IMC:", round(imc, 2), "\n")
}

return(round(imc, 2))
}

# Usar la función
calcular_imc(70, 1.75, "Ana")
calcular_imc(65, 1.68, "Luis")
calcular_imc(80, 1.82, "María")
Buenas Prácticas para Funciones:
  • Una tarea: Cada función debe hacer una sola cosa bien
  • Nombres descriptivos: calcular_promedio() vs calc()
  • Validar inputs: Verificar que los parámetros sean correctos
  • Documentar: Explicar qué hace, qué recibe y qué devuelve
  • Valores default: Parámetros opcionales con valores por defecto
Manejo Seguro de Datos

Protege tus datos y tu análisis con prácticas defensivas.

✅ Buenas Prácticas:
# 1. Verificar que el archivo existe
archivo <- "datos.csv"
if (!file.exists(archivo)) {
stop("Archivo no encontrado: ", archivo)
}

# 2. Hacer backup de datos originales
datos_original <- read.csv(archivo)
datos_trabajo <- datos_original # Copia para trabajar

# 3. Verificar estructura de datos
stopifnot(
"edad" %in% names(datos_trabajo),
nrow(datos_trabajo) > 0,
is.numeric(datos_trabajo$edad)
)

# 4. Manejar valores faltantes explícitamente
if (any(is.na(datos_trabajo$edad))) {
cat("Encontrados", sum(is.na(datos_trabajo$edad)), "valores NA en edad\n")
# Decidir estrategia: eliminar, imputar, etc.
}

# 5. Validar resultados
resultado <- mean(datos_trabajo$edad, na.rm=TRUE)
if (resultado < 0 || resultado> 120) {
warning("Edad promedio sospechosa: ", resultado)
}
