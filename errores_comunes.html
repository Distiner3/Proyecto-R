Guía de Troubleshooting para R

¡No te preocupes! Todos los programadores enfrentan errores. Esta guía te ayudará a identificar y solucionar los errores más comunes en R. Aprende a leer los mensajes de error y encuentra soluciones rápidas.

Consejo Pro: Los errores son oportunidades de aprendizaje. Cada error que resuelves te hace mejor programador.

Errores de Objetos y Variables
Muy Común
Error: "object 'x' not found"
⭐ Uno de los errores más frecuentes para principiantes
Error: object 'datos' not found

¿Qué significa? R no puede encontrar el objeto (variable) que estás intentando usar.

Causas comunes:

  • No has creado la variable aún
  • Error en el nombre (mayúsculas/minúsculas)
  • La variable está en otro entorno
  • Escribiste mal el nombre
Soluciones:

1. Verificar que la variable existe:

ls() # Ver todos los objetos en memoria
exists("datos") # Verificar si 'datos' existe

2. Crear la variable si no existe:

datos <- read.csv("mi_archivo.csv") # O como sea apropiado

3. Verificar ortografía (R distingue mayúsculas):

❌ Datos # Incorrecto si creaste 'datos'
✅ datos # Correcto
Prevención

Usa nombres descriptivos y consistentes. Siempre verifica que tus variables se crearon correctamente después de ejecutar el código.

Común
Error en nombres de columnas
Error in datos$edad : $ operator is invalid for atomic vectors

¿Qué significa? Estás intentando acceder a una columna de algo que no es un data frame.

Soluciones:

1. Verificar el tipo de objeto:

class(datos) # Debe ser "data.frame"
str(datos) # Ver estructura completa

2. Si es un vector, usa índices:

❌ datos$edad # No funciona con vectores
✅ datos[1] # Primer elemento del vector

3. Ver nombres de columnas disponibles:

names(datos) # Ver nombres de columnas
colnames(datos) # Alternativa
Errores de Funciones
Muy Común
Error: "could not find function"
Error: could not find function "ggplot"

¿Qué significa? R no reconoce la función que estás intentando usar.

Soluciones:

1. Instalar el paquete necesario:

install.packages("ggplot2") # Instalar paquete
library(ggplot2) # Cargar paquete

2. Verificar ortografía de la función:

❌ gplot() # Función inexistente
✅ ggplot() # Función correcta

3. Verificar paquetes cargados:

search() # Ver paquetes cargados
Solución Rápida: Si ves este error, casi siempre necesitas hacer library(nombre_paquete) antes de usar la función.
Común
Error en argumentos de funciones
Error in mean(datos, edad) : 2 arguments passed to 'mean' which requires 1

¿Qué significa? Estás pasando más (o menos) argumentos de los que la función espera.

Soluciones:

1. Verificar la sintaxis correcta:

❌ mean(datos, edad) # Incorrecto
✅ mean(datos$edad) # Correcto

2. Consultar la ayuda de la función:

?mean # Ver documentación
help(mean) # Alternativa

3. Ver ejemplo de uso:

example(mean) # Ver ejemplos
Errores de Sintaxis
Muy Común
Error: "unexpected symbol/character"
Error: unexpected symbol in "datos <- read.csv"archivo.csv""

¿Qué significa? Hay un error de sintaxis en tu código.

Soluciones Comunes:

1. Verificar paréntesis y comillas:

❌ datos <- read.csv"archivo.csv" # Falta paréntesis
✅ datos <- read.csv("archivo.csv") # Correcto

2. Comillas mal cerradas:

❌ nombre <- "Juan # Comilla no cerrada
✅ nombre <- "Juan" # Correcto

3. Operadores mal escritos:

❌ x < - 5 # Espacios en el operador
✅ x <- 5 # Correcto
Prevención

Usa un editor que resalte la sintaxis (como RStudio). Los colores te ayudarán a identificar errores de sintaxis rápidamente.

Común
Error por comas faltantes
Error: unexpected symbol in "c(1 2 3)"
Solución:
❌ c(1 2 3) # Faltan comas
✅ c(1, 2, 3) # Correcto
❌ data.frame(x=c(1,2) y=c(3,4)) # Falta coma
✅ data.frame(x=c(1,2), y=c(3,4)) # Correcto
Errores con Datos
Muy Común
Error: "cannot open file"
Error in file(file, "rt") : cannot open the connection
Warning: cannot open file 'datos.csv': No such file or directory

¿Qué significa? R no puede encontrar el archivo que intentas abrir.

Soluciones:

1. Verificar directorio actual y archivos:

getwd() # Ver directorio actual
list.files() # Ver archivos disponibles

2. Cambiar al directorio correcto:

setwd("C:/ruta/a/tus/datos") # Cambiar directorio

3. Usar ruta completa:

datos <- read.csv("C:/ruta/completa/datos.csv")

4. Verificar nombre del archivo:

❌ "datos.csv" # Si el archivo se llama "Datos.csv"
✅ "Datos.csv" # Nombre exacto
Tip Rápido: En RStudio, puedes usar el panel "Files" para navegar y hacer clic en "More → Set As Working Directory".
Común
Problemas con valores NA (faltantes)
Warning: NAs produced by coercion
Soluciones:

1. Identificar valores NA:

is.na(datos) # Ver dónde hay NA
sum(is.na(datos)) # Contar NA
complete.cases(datos) # Filas completas

2. Remover NA en cálculos:

❌ mean(datos$edad) # Devuelve NA si hay faltantes
✅ mean(datos$edad, na.rm = TRUE) # Ignora NA

3. Limpiar datos:

datos_limpios <- na.omit(datos) # Remover filas con NA
Errores en Gráficos
Ocasional
Problemas con ventana de gráficos
Error in plot.new() : figure margins too large
Soluciones:

1. Agrandar ventana de gráficos en RStudio

2. Reiniciar dispositivo gráfico:

dev.off() # Cerrar dispositivo gráfico actual

3. Ajustar márgenes:

par(mar = c(5, 4, 4, 2)) # Ajustar márgenes
Ocasional
Error: "invalid color name"
Solución:

Ver colores disponibles:

colors() # Ver todos los nombres de colores
# O usar códigos hexadecimales
plot(x, y, col = "#FF5733")
❌ plot(x, y, col = "azul") # No existe "azul"
✅ plot(x, y, col = "blue") # Correcto
Consejos Generales para Depurar Código
Estrategias de Debugging
  • Lee el mensaje completo: Los errores suelen indicar la línea exacta del problema
  • Ejecuta línea por línea: Encuentra exactamente dónde ocurre el error
  • Verifica tipos de datos: Usa class() y str()
  • Simplifica: Prueba con datos más pequeños primero
Herramientas Útiles
  • traceback() - Ver dónde ocurrió el error
  • browser() - Pausar ejecución y explorar
  • debug(función) - Debuggear función específica
  • options(error = recover) - Debugging avanzado
Recuerda

Los errores son normales y útiles. Cada error que resuelves te ayuda a entender mejor R. No te frustres, ¡incluso los programadores expertos consultan estos errores regularmente!
